PART D — Phase 3 QA

Run a full QA pass.

Verify flows:

1. Free user uses route engine.
2. Free user hits premium limit and sees upgrade.
3. User starts premium checkout.
4. Mock/sandbox M-Pesa success activates premium.
5. User buys one-off pack.
6. User creates consultation request.
7. Professional responds with offer.
8. User grants limited data access.
9. User pays for consultation.
10. Professional sees only allowed context.
11. Notifications appear.
12. Admin can inspect payment and consultation state.
13. Duplicate payment callback does not double-unlock.
14. Failed payment does not unlock features.
15. User cannot view another user's payment, journal, portfolio, or consultation.

Security/privacy checks:

* no M-Pesa PIN collected
* no bank passwords collected
* no broker credentials collected
* no exact values shown when hidden/range mode is selected
* professional access is scoped
* data grants can expire/revoke
* payment webhooks are idempotent
* secrets are env-based
* logs do not expose sensitive fields
